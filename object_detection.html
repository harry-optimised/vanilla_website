Object Detection with Deep Learning Series

This series dives deep into the technical detail behind the early objection detection architectures using Deep Learning. I cover the canonical two-stage object detectors and cover the transition to single-stage detectors which paved the way for the current modern architectures like YOLOv8 and more.


  • Part 1: R-CNN

    We'll start our journey into object detection with deep learning by exploring how the object detection problem can be framed as region proposal and classification. R-CNN (Regions with CNN Features) was the first architecture to use deep learning for some of the system components. This article examines the architecture in detail, and highlights the various tricks that made it effective.


  • Part 2: Spatial Pyramid Pooling

    This article explores the SPP-Net model, the architecture that introduced the spatial pyramid pooling layer. This is a nice model to study because it tackles one of the key problems with R-CNN, which is that it is slow. It also paves the way towards Fast R-CNN.


  • Part 3: Fast R-CNN

    Fast R-CNN addressed the slowness of R-CNN by moving everything into one network. They use a variation of the spatial pyramid pooling layer (although they call it ROI pooling) to allow the network to sample features at different scales. Finally we discuss how bounding boxes are parameterised within a network, as it's a good example of how to design your network to match the problem.


  • Part 4: Faster R-CNN

    Faster R-CNN innovated in the network architecture. They recognised that region proposal was also object detection, so they put an object detector in an object detector. This article spends some time exploring how to think of the architecture as building blocks, which is a useful way to think about object detectors in general. I also introduce anchors as a way of parameterising a set of initial bounding boxes.


  • Part 5: Mask R-CNN

    As a roundup of two-stage object detectors, we explore Mask R-CNN, an envolution of Faster R-CNN that adds a mask branch to the network. This is a good example of how to add additional branches to a network to give it more capabilities. It also shows how to use the same feature maps for multiple tasks. I'll spend most of the article discussing the ROI Align layer which is essential for predicting accurate masks.


  • Part 6: Deep Multi Box

    This is the first article in the single stage detector mini series. It's a good one to start with because it introduces the concept of framing object detection as regression of box coordinates. Multibox is an early incarnation of YOLO and SSD, it uses bipartite matching to assign ground truth boxes to anchor boxes.


  • Part 7: YOLO v1

    In this article we look at the YOLO v1 model. This is a good example of cleverly choosing what the network outputs should represent, and then designing a loss function to achieve that goal. YOLOv1 is actually quite simple once you know how it works. I'll spend a lot of time on the loss function in this article.


  • Part 8: Single Shot Detector

    This article takes a brief digression to look at the Single Shot Detector (SSD). I explore this paper because it ties the concepts of one-stage and two-stage architectures quite nicely.


  • Part 9: YOLO Version 2 and 3

    We wrap this series for the time being with a bag of tricks that were applied to the YOLO model to produce YOLOv2 and YOLOv3. These are general orthogonal improvements that can be applied to all object detectors. It's a good example of lots of little tricks that stack up.
